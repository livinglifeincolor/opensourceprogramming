# 풀스택 웹 개발 학습 로드맵

이 프로젝트(프론트엔드: Next.js, 백엔드: Flask, 데이터베이스: PostgreSQL, 인프라: Docker)를 바탕으로 웹 개발을 공부하는 추천 순서입니다.

## 1단계: 전체 구조 파악 및 인프라 (큰 그림 보기)

코드를 수정하기 전에 이 서비스들이 어떻게 유기적으로 연결되어 돌아가는지 이해하는 것이 좋습니다. 현대 백엔드 개발에서 필수적인 컨테이너 기술을 이해하는 단계입니다.

- **서버 환경 구축 분석 (`docker-compose.yml`)**:
  - **서비스 구성**: `db`, `backend`, `frontend` 세 가지 서비스가 어떻게 구성되고 포트가 연결되는지 확인하세요.
  - **환경 변수**: 백엔드 서버가 DB에 접속하기 위해 필요한 정보(DB_HOST, DB_NAME 등)가 어떻게 전달되는지 파악하세요.
- **직접 실행해보기**: `docker compose up --build` 명령어로 직접 띄워보고, 웹 브라우저로 `localhost:3000`에 접속해 이것저것 클릭(회원가입, 글쓰기 등)해 봅니다.

## 2단계: 데이터베이스 & 백엔드 (데이터 모델링과 서버 로직)

눈에 보이지 않는 뒷단(Backend)부터 시작해 '데이터가 어떻게 다뤄지는지'를 먼저 이해하는 것이 좋습니다.

### 2-1. 데이터 모델링 이해 (`backend/schema.sql`)

백엔드 개발의 시작은 데이터가 어떻게 저장되는지 파악하는 것입니다.

- **사용자(users) 테이블**: 아이디, 이메일, 암호화된 비밀번호(`password_hash`)가 어떻게 정의되어 있는지 확인하세요.
- **게시글(posts) 테이블**: 게시글이 어떤 정보를 담고 있으며, 특히 `user_id`를 통해 어떻게 사용자 테이블과 관계(Foreign Key)를 맺고 있는지 분석하세요.

### 2-2. 데이터베이스 연동 로직 (`backend/db.py`)

코드가 실제 데이터베이스와 대화하는 방식을 배울 수 있습니다.

- **연결 관리**: `get_db()` 함수가 어떻게 세션을 관리하고, `close_db()`가 요청 종료 시 연결을 닫는지 확인하세요.
- **쿼리 실행**: `query_db()` 함수를 통해 SQL 문이 실행되고 결과가 딕셔너리 형태로 반환되는 과정을 분석하세요.

### 2-3. 비즈니스 로직 및 API 분석 (`backend/app.py`)

가장 핵심적인 부분으로, 클라이언트의 요청을 처리하는 상세 로직을 학습합니다.

- **회원가입/로그인**:
  - 비밀번호를 그대로 저장하지 않고 `generate_password_hash`로 암호화하는 이유를 생각하며 코드를 보세요.
  - 로그인 성공 시 `session["user_id"]`를 통해 세션 기반 인증이 어떻게 이루어지는지 확인하세요.
- **게시판 CRUD**:
  - **C(Create)**: 세션에서 사용자 정보를 가져와 글을 작성하는 과정.
  - **R(Read)**: 전체 목록을 가져올 때 `LIMIT`와 `OFFSET`을 이용한 페이지네이션 구현 방식.
  - **U/D(Update/Delete)**: 수정이나 삭제 시, 현재 로그인한 사용자가 글 작성자인지 확인하는 권한 체크 로직을 반드시 확인하세요.

## 3단계: 프론트엔드 (화면 구성과 사용자와의 상호작용)

백엔드에서 데이터를 어떻게 주는지 알았다면, 화면(Frontend)에서 그걸 어떻게 받아서 예쁘게 그리는지 살펴봅니다.

- **`frontend/src/lib/api.ts`**: 프론트엔드가 백엔드 API(방금 app.py에서 본 주소들)를 어떻게 호출하는지 확인합니다. Fetch API의 사용법을 익힐 수 있습니다.
- **`frontend/src/app/layout.tsx` & `page.tsx`**: Next.js 프로젝트의 진입점과 전체 페이지를 감싸는 레이아웃(네비게이션 바 등)을 이해합니다.
- **`frontend/src/app/posts/page.tsx`**: 백엔드에서 받아온 '게시글 목록' 데이터를 React 컴포넌트가 어떻게 화면에 그려주는지(렌더링) 분석합니다.

## 4단계: 하나의 기능 디버깅하듯 따라가 보기 (Data Flow)

단편적인 이해를 넘어, 전체 흐름을 관통해 봅니다.
예를 들어 **"게시글 작성"** 이라는 하나의 기능이 처음부터 끝까지 어떻게 동작하는지 추적해 봅니다.

1. 사용자가 글을 쓰고 전송 버튼을 누른다. (`frontend/src/app/posts/write/page.tsx`)
2. 프론트엔드 API 함수가 호출된다. (`frontend/src/lib/api.ts`)
3. 백엔드 라우터가 요청을 받는다. (`backend/app.py` 의 `POST /api/posts` 부분)
4. 데이터베이스에 정보를 INSERT 한다. (`backend/db.py`)

## 5단계: 직접 실습하며 확장하기 (Hands-on)

읽기만 해서는 내 것이 되지 않습니다. 코드를 읽는 것에서 나아가, 아래 기능들을 직접 추가해 보며 실력을 키워보세요.

- **난이도 하**: 프론트엔드 화면 디자인/색상 살짝 바꿔보기. **예외 처리 강화**: 현재 코드에서 발생할 수 있는 다양한 오류 상황(예: DB 연결 실패)에 대한 응답 세분화.
- **난이도 중**: 게시글 테이블(`posts`)에 **"조회수(Views)"** 컬럼을 추가하고, 상세 보기 API가 호출될 때마다 조회수를 1씩 증가시켜 화면에 표시하기. **검색 기능**: 게시글 목록 API에서 제목이나 내용으로 검색할 수 있는 쿼리 파라미터 추가.
- **난이도 상**: **"댓글(Comments)" 기능 구현**. DB에 comments 테이블 추가 -> 백엔드 API 작성 -> 프론트엔드 UI 및 API 연동.
