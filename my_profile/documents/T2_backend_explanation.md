# 1. 데이터 모델링 이해 (`schema.sql`)

백엔드 개발의 시작은 데이터의 구조를 설계하는 것입니다. 이 프로젝트는 PostgreSQL을 사용하며 두 개의 테이블로 구성되어 있습니다.

* **사용자 테이블 (`users`)**:
  * `id`: 각 사용자를 구별하는 고유 번호(Primary Key)입니다.
  * `username`, `email`: 중복을 허용하지 않는(`UNIQUE`) 사용자 정보입니다.
  * **보안**: 비밀번호를 평문으로 저장하지 않고 암호화된 `password_hash` 형태로 저장하도록 설계되었습니다.


* **게시글 테이블 (`posts`)**:
  * `title`, `content`: 글의 제목과 본문입니다.
  * **관계 연동 (`user_id`)**: `REFERENCES users(id)`를 통해 어떤 사용자가 글을 썼는지 연결합니다.
  * `ON DELETE CASCADE`: 사용자가 탈퇴(삭제)되면 그 사용자가 쓴 글도 자동으로 삭제되도록 설정되어 있습니다.


# 2. 데이터베이스 연동 로직 (`db.py`)

파이썬 코드(`Flask`)가 데이터베이스와 어떻게 데이터를 주고받는지 정의합니다.

* **연결 관리 (`get_db`)**: `os.environ.get`을 사용하여 도커 환경 변수에서 DB 접속 정보를 가져와 연결을 생성합니다.
* **자원 해제 (`close_db`)**: 클라이언트의 요청이 끝나면 사용한 DB 연결을 자동으로 닫아 자원 낭비를 방지합니다.
* **쿼리 실행 (`query_db`)**: SQL 문을 실행하고 그 결과를 파이썬의 딕셔너리(`Dictionary`) 형태로 반환하여, 코드에서 데이터에 쉽게 접근할 수 있도록 돕습니다.


# 3. 비즈니스 로직 및 API 분석 (`app.py`)

클라이언트(프론트엔드)의 요청에 따라 실제 기능을 수행하는 핵심 로직입니다.

* **인증 및 보안 (Auth)**:
  * **회원가입**: `generate_password_hash`를 사용해 비밀번호를 안전하게 암호화하여 저장합니다.
  * **로그인 및 세션**: `check_password_hash`로 비밀번호를 확인하고, 성공 시 `session["user_id"]`에 사용자 정보를 저장하여 이후 요청에서도 로그인 상태를 유지합니다.


* **게시판 기능 (CRUD)**:
  * **목록 조회 (Read)**: `LIMIT`와 `OFFSET`을 사용하여 페이지별로 10개씩 글을 가져오는 페이지네이션이 구현되어 있습니다.
  * **권한 체크 (Update/Delete)**: 글을 수정하거나 삭제할 때, `session`의 `user_id`와 게시글의 `user_id`를 비교하여 본인의 글인지 확인하는 권한 로직이 포함되어 있습니다.
